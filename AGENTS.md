# 项目级 AGENTS 规则

## 优先级
1. 用户当前明确指令
2. 本文件
3. 全局 AGENTS

## 沟通
- 默认使用中文
- 先结论，后依据
- 信息不足时明确不确定性，不臆测

## 执行
- 最小可行改动，只改与需求直接相关的内容
- 先定位根因，再改动
- 不引入无关功能或重构
- 每次修改需说明：修改点、影响范围、验证方式

## 工程约束
- 文件统一 UTF-8
- 失败路径显式返回，不伪造成功
- 注释只写关键意图，使用中文

## 命令边界
- Linux 使用 bash 语法
- 未经用户明确同意，不执行编译、测试、打包、部署等高成本命令
- 删除操作、批量替换、结构性重构、数据库写操作需先确认

## 数据库约束（若后续涉及）
- 默认 PostgreSQL，schema 为 `mc`
- 默认只读，写操作需先给 SQL 与影响说明并确认
- 查询前先确认表结构与字段含义
- 单次查询默认不超过 50 条
